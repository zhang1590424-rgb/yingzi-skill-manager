# Components Library

This folder is a research library for reusable component ideas, motion patterns, hero systems, layout modules, and interface primitives gathered from strong public sources.

## Files

- `gsap-explore.md` - 34 official GSAP demo references with demo links, CodePen links, likely APIs, and clean-room implementation notes.
- `aceternity-ui.md` - large Aceternity UI component index for hero systems, backgrounds, cards, and motion-heavy surfaces.
- `21st-community.md` - discovery pages plus practical community primitives and richer community references from 21st.
- `reactbits.md` - React Bits index for text animation, motion effects, composed components, and atmosphere/background patterns.

## How to use this library

- Start with the active style skill, not with the component library.
- Pick one or two component patterns that strengthen the current style instead of dumping many borrowed ideas into one page.
- Rebuild the pattern so it matches the style's typography, motion level, color logic, density, and spacing rules.
- Prefer larger image-led or structure-led components over tiny decorative widgets.
- Keep motion tasteful: hero reveal, sectional scroll entry, hover states, and one or two memorable interactions are usually enough.
- Do not paste third-party code wholesale into taste outputs; use the source links to inspect original code when needed and then adapt the idea cleanly.

## Fast source mapping

- GSAP Explore: best for scroll scenes, cursor behavior, gallery motion, text reveals, and advanced interaction choreography.
- Aceternity UI: best for motion-heavy hero modules, glowing surfaces, nav experiments, backgrounds, and showcase sections.
- 21st Community: best for practical primitives, production UI pieces, and newer community hero/shader ideas.
- React Bits: best for text animation, micro-interactions, cursor effects, animated atmospheres, and composed React modules.
