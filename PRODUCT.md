# 影子产品约束

## 用户

主要用户是老张：互联网产品经理，使用 Codex 开发产品和管理个人知识资产，不希望被底层路径、软链接和 Agent 差异拖慢。

## 核心任务

影子解决三个问题：

- 把本地 Skill 主库作为唯一事实源。
- 看清 Skill 在 Trae CN、Codex、Claude Code、Aiden、Agents 以及手动添加项目中的启用状态。
- 通过软链接把主库 Skill 分发、收回或覆盖到目标位置。

## 命名隐喻

「影子」表示一个实体在多个地方生成影子：主库 Skill 是唯一事实源，Agent 和项目目录里的软链接是它在不同工作环境中的影子。

## 不做什么

- 不做营销首页。
- 不做通用 Skill 市场浏览、推荐和评价。
- 不做云同步、多账号、自动扫描所有项目。
- 第一版不做覆盖前备份和操作日志。

## 产品判断

每个界面决策都先回答：

- 用户当前想判断什么状态？
- 下一步最可能执行什么操作？
- 这个操作是否会改变主库或覆盖目标？
- 是否能在不理解技术细节的情况下做出安全判断？
