# 全局交互动效收敛设计

日期：2026-05-23

## 1. 背景

当前页面在用户操作后有多处状态变化是瞬时完成的：点击 Skill 后右侧详情栏直接出现，Skill 行 hover 后操作按钮突然替换状态信息，技能组合成员展开、抽屉、弹窗、批量操作条和提示条也缺少进入过渡。

问题本质不是“缺少装饰动画”，而是界面没有告诉用户变化从哪里来、影响了哪块空间。对本地工作台来说，动效应该帮助建立空间连续性；同时「影子」本身也有一定炫技属性，动效可以更有展示感，但必须服务于对象关系和操作反馈。

## 2. 核心判断

| 方案 | 判断 | 为什么 | 对用户的影响 |
|---|---|---|---|
| 只给详情侧栏加动画 | 不采用 | 用户感知到的生硬来自多处状态变化，单点处理会继续割裂 | 点击 Skill 会好一些，但 hover、弹窗和展开仍突兀 |
| 引入动画库统一管理 | 不采用 | 当前交互以 CSS 可表达的进入、显隐和布局过渡为主，引库会增加复杂度 | 维护成本更高，收益不足 |
| 建立有展示感的 CSS 动效层 | 采用 | 用统一时间、曲线、轻微弹性和关键帧覆盖主要状态变化，改动集中且可降级 | 页面变化更有空间感和质感，但不影响操作效率 |

## 3. 动效规则

- 时间控制：高频 hover 使用约 140ms，布局和面板进入使用约 240ms，抽屉/弹窗使用约 260ms。
- 空间方向：右侧详情和抽屉从右侧滑入并带轻微阴影增强，弹窗从中心轻微缩放进入，工具条和提示条从下方弹出，展开内容向下展开。
- 行内操作：默认保留稳定占位，hover / focus 时用透明度、位移、缩放和按钮错峰进入替换状态信息，不改变行高、不遮挡内容。
- 展示感边界：允许轻微弹性、阴影、亮面扫过和层级位移；不使用紫色渐变、玻璃拟态、发光边框或持续循环动效。
- 展开内容：技能组合成员、路径详情、正文预览用高度或进入动画表达层级变化。
- 降级规则：继续使用 `prefers-reduced-motion: reduce` 关闭动画和过渡。

## 4. 覆盖范围

- 技能列表：列表详情栏开合、行内操作显隐、列表行 hover / selected 反馈。
- 技能组合与应用工作台：组合成员展开、转移列 drop target、转移行选择和 hover。
- 覆盖式界面：分发抽屉、问题处理抽屉、导入弹窗、组合编辑弹窗、确认弹窗。
- 临时反馈：批量选择条、toast、导入存量工具条、筛选条进入。
- 内联展开：路径详情、正文预览展开、预检区状态变化。

## 5. 验证标准

- 点击 Skill 后右侧详情栏不再瞬时跳出，列表宽度和详情内容有连续过渡。
- hover Skill 行时，状态信息和三个操作按钮柔和切换，不遮挡名称、描述和标签。
- 打开抽屉、弹窗、批量操作条和 toast 时有轻量进入过渡。
- 展开技能组合成员和正文预览时不出现明显跳变。
- `prefers-reduced-motion` 下动画被关闭。
- `npm run build` 通过。
- `cargo check --manifest-path src-tauri/Cargo.toml` 通过。
