# Skill 压缩包导入设计

日期：2026-05-23

## 1. 背景

原导入入口只支持选择包含 `SKILL.md` 的文件夹。实际使用中，Skill 经常以压缩包形式从聊天、仓库导出或跨设备传递。如果必须先手动解压，用户需要理解目录结构，还容易选错层级。

## 2. 设计决策

| 决策 | 为什么 | 对用户的影响 |
|---|---|---|
| 一个导入弹窗承载文件夹和 `.zip` | 导入本质是把本地 Skill 放入主库，不需要做来源切换；系统选择器无法可靠混选文件和目录 | 用户仍从同一个「导入技能」入口进入，再按实际素材选择文件夹或压缩包 |
| 压缩包只支持 `.zip` | macOS 和常见分享链路对 `.zip` 支持最稳定，其他格式会扩大边界 | 遇到 `.tar` 等格式时先手动转换，第一版行为更可预期 |
| `.zip` 可包含单个或多个 Skill | 单个分享和批量迁移都是真实场景 | 用户可以一次导入一个 Skill，也可以导入一个 Skill 集合 |
| 导入前完整校验 | 半截导入会让主库状态难解释 | 同名冲突、无有效 Skill、非法路径会整体失败 |
| 导入后不自动分发 | 入库和启用仍是两个不同决策 | 新 Skill 先进入技能列表，再由用户选择分发目标 |

## 3. 交互规格

```text
点击「导入技能」
→ 弹窗展示一个本地选择区
→ 选择「文件夹」或「.zip」
→ 打开对应的系统选择器
→ 文件夹：要求自身包含 SKILL.md
→ .zip：根目录包含 SKILL.md 时作为单个 Skill，否则扫描一级子目录中的 Skill
→ 复制到 ~/.skills-manager/skills
→ 关闭弹窗并回到技能列表
```

文案：

- 主文案：`点击选择技能文件夹或 .zip`
- 辅助文案：`文件夹需包含 SKILL.md；.zip 可包含单个或多个技能`
- 补充文案：`导入后进入主库，不会自动分发`
- 动作：`选择文件夹`、`选择 .zip`

## 4. 错误处理

- 非 `.zip` 文件：提示只支持 `.zip` 压缩包。
- 文件夹缺少 `SKILL.md`：提示不能作为正式技能入库。
- `.zip` 内没有有效 Skill：提示压缩包里未找到 `SKILL.md`。
- `.zip` 内包含绝对路径或 `..`：拒绝导入，避免解压越界。
- 主库已有同名 Skill：整体失败，沿用主库同名冲突提示。

## 5. 验收标准

- 导入弹窗不增加来源切换。
- 同一个导入弹窗可以导入文件夹或 `.zip`。
- 单个 Skill 压缩包和多个 Skill 压缩包都能进入主库。
- 导入后仍不自动分发到 Agent 或项目。
- `npm run build`、`cargo check --manifest-path src-tauri/Cargo.toml` 通过。
