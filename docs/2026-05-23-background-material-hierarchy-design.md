# 背景材质层级设计

日期：2026-05-23

## 1. 背景

当前「影子」已经形成了冷中性浅灰、三栏工作台、轻列表行的基础语言。但初始化欢迎页、初始化左侧进度栏和主应用左侧栏仍然偏实体化，视觉存在感接近当前操作区。

这会带来一个问题：用户当前真正要处理的是右侧任务内容或主工作区，但非当前关注区域仍在抢注意力。

本次设计目标不是增加玻璃拟态装饰，而是建立一套可复用的背景材质层级：让非焦点区域退后，让焦点区域保持清晰。

## 2. 核心决策

| 决策 | 为什么 | 对用户的影响 |
|---|---|---|
| 用材质层级替代装饰型玻璃拟态 | 项目定位是本地工作台，不是展示型页面；玻璃拟态容易降低可读性 | 界面更安静，长期使用不累 |
| 透明只用于非焦点区域 | 当前操作区需要稳定的阅读和点击反馈 | 用户能明确知道现在该看哪里、点哪里 |
| 初始化和主工作台共享同一层级规则 | 首次体验和日常使用不能像两个产品 | 视觉连续，产品气质更统一 |
| 弹窗和抽屉背后内容退后 | 模态任务需要明确焦点 | 用户处理确认、导入、分发时更不容易分心 |

## 3. 材质层级

| 层级 | 用途 | 视觉规则 |
|---|---|---|
| `base` | 应用底色 | 冷中性浅灰，接近当前 `--app-bg` |
| `quiet-material` | 非当前关注区域，如侧边栏、初始化 rail、欢迎页装饰面 | 半透明浅灰 / 白、轻微 backdrop blur、低对比边框 |
| `focus-surface` | 当前操作区、列表、表单、按钮、弹窗主体 | 接近实色，文字和边框对比稳定 |
| `modal-scrim` | 弹窗、抽屉后的背景退后层 | 轻 dim + blur，不压暗成网页式遮罩 |

透明效果必须服务注意力，不做独立装饰。

## 4. 区域方案

### 4.1 初始化欢迎页

欢迎页保留「影子」品牌门面，但降低右侧影子层场的视觉强度：

- 背景使用冷中性浅灰到浅白的细微层次。
- 右侧影子平面降低透明度、边框和阴影强度。
- 主标题、标语和「开始探索」按钮保持清晰实色。
- 不增加彩色光、发光边框、漂浮粒子或装饰球。

### 4.2 初始化左侧进度栏

初始化 rail 是流程位置提示，不是当前任务区：

- 使用 `quiet-material`。
- 背景比主任务区更透明、更低对比。
- active 步骤可以变清晰，但不要大面积高亮。
- 已完成步骤只保留轻状态，不抢当前步骤。

### 4.3 主应用左侧栏

左侧栏是导航和范围入口，应保持可识别但退到背景：

- 侧边栏背景改为克制半透明材质。
- 当前 active 导航项保留明确白底和左侧短线。
- hover 项短暂变清晰，帮助定位。
- 底部元信息继续低对比展示，不扩展成视觉模块。

### 4.4 弹窗和抽屉背景

弹窗和抽屉打开时，背景内容需要退后：

- backdrop 使用轻 dim + blur。
- 浮层主体保持实色，不透明。
- toast 层级仍高于遮罩和浮层。
- 背景退后不应造成文字残影、闪烁或明显性能问题。

## 5. 不做范围

- 不做全应用玻璃化。
- 不把列表、表单、按钮、弹窗主体改成透明。
- 不增加紫色渐变、彩色光、发光边框、bokeh、漂浮装饰。
- 不重构布局和信息架构。
- 不引入新的 UI 库。

## 6. 实现建议

优先通过 CSS token 和现有 class 调整：

- 新增或明确 `--material-quiet`、`--material-quiet-border`、`--scrim-bg` 等语义变量。
- 调整 `.sidebar`、`.onboarding-shell.welcome`、`.onboarding-rail`、`.shadow-plane`、`.drawer-backdrop`、`.dialog-backdrop`。
- 对支持 blur 的区域使用 `backdrop-filter`，并保证无 blur 时仍有可接受的背景色。
- 动效只使用 120-160ms 的 background、border、opacity 变化。

## 7. 验证标准

实现后必须验证：

- `npm run build`
- `cargo check --manifest-path src-tauri/Cargo.toml`
- 桌面应用能启动。
- 初始化欢迎页、初始化流程、主工作台、弹窗和抽屉视觉连续。
- 当前操作区文字清晰，按钮和表单对比足够。
- 左侧栏和初始化 rail 明显退后，但不影响识别和导航。
- 常见桌面窗口尺寸下不重叠、不溢出。
