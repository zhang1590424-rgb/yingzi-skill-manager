# 空状态引导与菜单命名设计

日期：2026-05-23

## 1. 背景

新用户第一次打开影子时，技能列表、技能组合、我的 Agent、我的项目都可能为空。当前空态偏功能占位，只告诉用户“没有数据”，没有解释下一步做什么，以及完成后能获得什么能力。

本次改动目标是把空状态变成进入工作流的起点：用户点击对应菜单后，主视觉直接引导添加或创建，并说明完成后能实现的效果。

## 2. 命名调整

| 原名称 | 新名称 | 为什么 | 对用户的影响 |
|---|---|---|---|
| 全局 Agent | 我的 Agent | 用户不是在理解全局目录，而是在管理自己正在使用的 Agent | 导航更像个人工作台，降低技术感 |
| 项目应用 | 我的项目 | 用户先关心项目是否纳入管理，再关心 Skill 如何应用 | 项目入口更直观，不把“应用”误解成最终动作 |

说明：底层概念仍保留 Agent、项目和软链接应用关系；本次只调整用户可见菜单与页面标题。

## 3. 空状态策略

| 场景 | 触发条件 | 主引导 | 说明文案方向 | 主按钮 |
|---|---|---|---|---|
| 技能列表为空 | 主库中没有 Skill | 导入第一个技能 | 导入后进入本地主库，之后可分发到 Agent 或项目 | 导入技能 |
| 技能组合为空 | 没有技能组合 | 创建第一个组合 | 创建后可把多个 Skill 作为能力单元批量应用 | 新建组合 |
| 我的 Agent 无可应用内容 | 当前 Agent 下没有可展示 Skill，且主库为空 | 先导入技能 | 导入后可在未应用 / 已应用之间管理软链接 | 导入技能 |
| 我的项目没有项目 | 项目列表为空 | 添加第一个项目 | 添加后可按项目管理各 Agent 的技能应用状态 | 添加项目 |
| 我的项目无可应用内容 | 已选中项目，但主库为空 | 先导入技能 | 导入后可把技能应用到这个项目下的 Agent | 导入技能 |

搜索或筛选导致的空结果不使用主视觉插画，只保留轻量文案，避免误导用户去新增数据。

## 4. 视觉资产

新增 4 张小型位图示意图，放在 `public/empty-states/`：

- `skills.png`：文件夹、`SKILL.md` 和本地主库线索。
- `compositions.png`：多个 Skill 节点汇入一个技能组合。
- `agents.png`：Agent 工作区和等待连接的 Skill 节点。
- `projects.png`：项目文件夹、多个 Agent 插槽和应用路径。

视觉原则：

- 低饱和浅灰工作台风格，匹配现有 Codex app 式界面。
- 精致但克制，不做营销插画、紫色渐变、玻璃拟态、发光边框。
- 不依赖图内文字表达含义，避免生成文字不稳定。
- 尺寸按小型空态图使用，页面内通过 CSS 控制最大宽度和高度。

## 5. 组件设计

新增可复用 `EmptyGuidance` 组件，负责主视觉空态：

- `illustration`：示意图路径。
- `title`：下一步动作导向标题。
- `description`：添加或创建后能实现什么。
- `action` / `onAction`：唯一主按钮。

保留现有 `EmptyState` 作为轻量空态，用于搜索无结果、详情无匹配等非新手引导场景。

## 6. 验证点

- 技能列表为空时，页面主视觉展示导入引导和插画。
- 技能组合为空时，页面主视觉展示新建组合引导和插画。
- 我的 Agent 在主库为空时展示导入引导，不出现两个空白栏。
- 我的项目没有项目时展示添加项目引导；已有项目但主库为空时展示导入引导。
- 左侧菜单和页面标题使用「我的 Agent」「我的项目」。
- 常见桌面窗口尺寸下插画、文案、按钮不重叠、不溢出。
