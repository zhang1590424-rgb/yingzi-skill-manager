# 影子品牌与图标设计

## 背景

产品正式中文名从「技能中枢」调整为「影子」。

「影子」表达的是：一个主库 Skill 作为实体，在不同 Agent 和项目目录中生成可收回、可覆盖、可验证的影子。这个名字比「技能中枢」更贴近软链接分发模型，也降低了“中心化管理后台”的笨重感。

## 设计目标

- 让用户一眼理解：同一个实体可以映射到多个位置。
- 保持本地桌面工具的安静、可信、低噪声气质。
- 不把“影子”做成神秘、监控、暗黑或赛博风格。
- 图标在 macOS Dock、窗口侧栏和网页 favicon 中都能识别。

## 图标方案

采用「S 形 Skill 实体 + 单个投影」：

- 主实体使用近黑的粗 `S` 形，面积控制在图标主体区域的约三分之二，代表主库中的正式 Skill。
- `S` 的线条需要更有艺术性：保留厚重识别，但允许端点和曲线有轻微动势。
- 单个低饱和灰色 `S` 形投影向右下方错位，代表 Skill 在目标 Agent 或项目中的影子。
- 背景使用白色或浅灰白圆角底，延续当前工作台的中性材质。
- 图标使用生图工具生成 512x512 PNG，不使用文字、发光、渐变文字或复杂装饰。

## 用户影响

- 产品名从“管理中心”转为“分身/映射”的心智，更贴近实际操作。
- 用户看到图标时能联想到“一个东西在多个地方生效”，而不是市场、仓库或普通文件夹。
- 侧栏品牌更短，给中文工作台留下更多扫描空间。

## 实现范围

- 应用内品牌名显示为「影子」。
- 浏览器标题、Tauri 窗口标题和 macOS 打包应用名改为「影子」。
- `public/app-icon.png` 与 `src-tauri/icons/icon.png` 使用同一张简洁生图图标。
- 保留内部包名、Rust crate 名和本地数据目录，避免因为品牌改名影响历史数据或构建缓存。

## 验证标准

- `npm run build` 通过。
- `cargo check --manifest-path src-tauri/Cargo.toml` 通过。
- 桌面应用能启动，侧栏显示「影子」和新图标。
- 常见桌面窗口尺寸下品牌区不重叠、不溢出。
