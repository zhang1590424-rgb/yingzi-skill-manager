# 影子产品介绍页设计

## 背景

「影子」需要一个轻量产品介绍页，用于对外说明产品、展示真实操作、提供下载入口，并降低未签名 Mac 应用首次打开时的安装阻力。

页面不是完整官网，也不是营销长页。第一版只承担三个任务：

- 让用户快速理解「影子」是什么。
- 通过演示视频建立真实产品感。
- 帮用户完成下载和首次打开授权。

## 设计方向

参考 `https://mole.fit/zh/` 的原生 Mac 工具介绍页风格：克制、直接、以真实产品信息为中心。

采用方向：

- 浅灰 / 近白背景。
- 产品 icon、真实演示视频、真实下载信息作为主要视觉资产。
- 小圆角、细边框、低阴影。
- 编号章节组织内容。
- 文案保持冷静、具体，不做大而空的 AI 营销表达。

不采用：

- 紫色渐变、玻璃拟态、发光边框。
- 复杂 FAQ、隐私政策、边界说明等长内容。
- 通用 SaaS 官网式 hero、客户 logo、价格区块。

对用户的影响：页面看起来像一个可信的本地 Mac 工具下载页，重点信息少、路径短，用户不需要在安装前阅读大量解释。

## 页面落点

第一版做成独立静态介绍页，不放进 Tauri 桌面应用路由。

推荐位置：

```text
public/intro/
```

页面可直接作为静态资源部署，也方便后续迁移到 GitHub Pages、个人站或 release 页面。

## 页面结构

### 顶部导航

内容：

- 左侧：影子 icon + `影子`
- 右侧：`下载`

行为：

- 点击 `下载` 滚动到下载与安装区。

### Hero

内容：

- 产品 icon。
- 产品名：`影子`
- 一句话简介：`一个管理本地 Skill 的 Mac 应用。`
- 补充说明：`建立唯一 Skill 源文件，并用软链接分发到不同 Agent 和项目，实现 Skill 按需分发、统一维护。`
- 主按钮：`下载影子`

对用户的影响：第一屏直接回答「这是什么」和「能不能下载」。

### 00 · 看见它

内容：

- 标题：`先看真实操作`
- 简短说明：`从主库到 Agent 和项目，影子把 Skill 的分发状态放在同一个本地工作台里。`
- 演示视频：`/Users/bytedance/项目/作品介绍/videos/product-operation-short-dynamic-web.mp4`

实现时应把视频复制到介绍页可访问的静态资源目录中，避免页面依赖项目外部路径。

视频展示要求：

- 默认静音。
- 展示播放控件。
- 桌面端宽度优先，移动端自适应。
- 不自动强行播放，避免打扰用户。

### 01 · 它做什么

用三个短条目说明核心能力：

1. `唯一 Skill 主库`
   - 正式 Skill 先进入本地主库，主库是唯一事实源。
2. `软链接分发`
   - 把 Skill 按需应用到 Codex、Claude Code、Trae CN、Aiden 或项目目录。
3. `状态一眼确认`
   - 看见哪些 Skill 已应用、未应用，哪些目标有问题。

对用户的影响：不用解释技术架构，也能理解影子的核心价值是「统一维护、按需分发、状态可见」。

### 02 · 下载与安装

内容分两段。

第一段：下载。

- 主下载按钮：`下载 DMG`
- 辅助信息：`测试版 · macOS 14+ · Apple Silicon`

下载文件第一版指向现有 release 产物：

```text
release/yingzi_0.1.0-test.1_arm64.dmg
```

实现时如需静态部署，应复制或链接到页面可访问位置。

第二段：安装引导。

步骤：

1. 打开 DMG，把 `影子.app` 拖入 `Applications`。
2. 首次打开如果被 macOS 拦截，打开 `系统设置 → 隐私与安全性`。
3. 在页面底部点击 `仍要打开`，再确认打开。

说明文案：

`影子当前是未签名测试版。macOS 会对未签名应用做首次打开保护；请只在你确认来源可信时继续。`

依据：Apple 官方说明提到 macOS Gatekeeper 会检查 Developer ID 签名和 notarization；未签名或未 notarize 的应用可能需要用户在系统设置中临时放行。

来源：

- https://support.apple.com/en-us/102445

### 底部

内容：

- `影子`
- `本地 Skill 管理，收放有据。`
- 版本和平台信息：`v0.1.0-test.1 · macOS 14+`

## 响应式与可访问性

- 桌面端主体宽度控制在约 960px 到 1120px。
- 移动端单列展示，按钮不换行溢出。
- 视频容器保持 16:9。
- 所有按钮和链接有清晰 hover / focus 状态。
- 文字不依赖低对比灰色。
- 安装步骤用编号，不用长段落。

## 验证

实现后验证：

- 页面能在本地直接打开或通过 Vite 静态服务访问。
- 产品 icon 正常显示。
- 演示视频正常加载、播放、暂停。
- 下载链接可访问 DMG。
- 桌面窗口和窄屏宽度下文本、按钮、视频不重叠、不溢出。
- `npm run build` 通过。

## 暂不做

- FAQ。
- 价格、购买、许可证。
- 多语言。
- 隐私政策和服务条款。
- 复杂的功能详情长页。
- 应用内路由入口。
