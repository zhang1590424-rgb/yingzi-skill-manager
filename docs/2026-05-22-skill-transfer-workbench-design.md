# Skill 双栏应用工作台设计

日期：2026-05-22

## 1. 背景

当前「技能中枢」的全局应用和项目应用仍然需要先进入抽象页面，再在中栏选择 Agent 或项目。这个结构能表达数据模型，但对高频操作不够直接。

本轮调整目标是把真实管理对象前置：全局 Agent 和手动添加的项目都直接出现在左侧菜单里。用户点击某个 Agent 或项目后，在主区域通过「未应用 Skill / 已应用 Skill」双栏完成应用或收回。

视觉样式以 `docs/2026-05-22-skill-application-kanban-style-design.md` 为准：两列轻量看板，不新增异常状态列。

## 2. 核心决策

| 维度 | 决策 | 为什么 | 对用户的影响 |
|---|---|---|---|
| 左侧导航 | 保留技能列表、技能组合、设置；把 Trae / Codex / Claude Code 下放为左侧菜单项；项目应用分组下列出所有项目 | 用户真正要管理的是具体 Agent 或项目，不是「全局应用」这个抽象页 | 少一次选择，进入对象后直接操作 |
| 项目维度 | 点击项目后不再区分 Agent，默认对该项目下所有 Agent 一起应用 / 收回 Skill | 项目级管理优先便捷，减少重复对三个 Agent 操作 | 一次拖拽即可完成项目整体启用或停用 |
| 主工作区 | Agent / 项目对象使用双栏：左侧未应用，右侧已应用；不再显示独立 Skill 详情栏 | 应用和收回本质是把 Skill 放入或移出当前对象，详情栏在这个场景里信息价值低 | 双栏空间更大，操作更聚焦 |
| 交互方式 | 支持拖拽、双击和箭头按钮 | 拖拽方便但不应成为唯一通道 | 鼠标、触控板、键盘用户都能完成操作 |
| 问题提示 | 冲突、未入库、失效、格式异常不单独成列，只在已应用侧卡片和详情中提示；仅目标 Skill 根目录不存在时按未应用处理 | 已占用目标位置的问题不能藏在未应用列表；但目标目录尚未创建不代表同名 Skill 已占用 | 第一次给 Agent 或项目启用 Skill 时可以直接应用，系统自动创建目标目录 |
| 桌面拖拽运行时 | Tauri 窗口关闭内置 `dragDropEnabled` | Tauri 默认的文件拖放系统会接管 WebView 拖拽事件，导致前端 HTML5 `dragover/drop` 不稳定或不可达 | 双栏内部拖拽能真正触发应用 / 收回；导入仍通过文件夹选择器完成 |

## 3. 信息架构

```text
技能中枢
  技能列表
  技能组合
  全局 Agent
    Trae
    Codex
    Claude Code
  项目应用
    手动添加的项目 A
    手动添加的项目 B
  设置
```

左侧是对象导航，不再把「全局应用」和「项目应用」作为可进入页面。它们只作为分组标题。

## 4. 双栏工作区

```text
┌──────────────────────────────────────────────────────┐
│ Codex                                        搜索/筛选 │
├──────────────────────────┬───────────────────────────┤
│ 未应用 Skill              │ 已应用 Skill               │
│ product-research   未应用 │ code-review        已应用  │
│ ui-ux-pro-max      未应用 │ old-skill          已应用  │
└──────────────────────────┴───────────────────────────┘
```

### 4.1 全局 Agent

| 列 | 内容 |
|---|---|
| 未应用 Skill | `disabled` |
| 已应用 Skill | `enabled`、`unmanaged`、`conflict`、`broken`、`invalid`；项目根路径不存在时显示为 `pathMissing` |
| 应用范围 | 当前 Agent 的全局 Skill 目录 |

说明：如果只是 Agent / 项目下的 Skill 根目录尚未创建，例如 `<项目>/.codex/skills` 不存在，应归入未应用并允许应用时自动创建目录。

### 4.2 项目

项目按全部 Agent 聚合。

| 项目内状态 | 项目层显示 |
|---|---|
| 所有 Agent 都已启用 | 已应用 |
| 所有 Agent 都未启用且无异常 | 未应用 |
| 部分 Agent 已启用 | 部分应用 |
| 任一 Agent 有冲突 | 冲突 |
| 任一 Agent 有未入库技能 | 未入库 |
| 项目路径不存在 | 路径不存在 |
| 任一 Agent 有失效或格式异常 | 路径异常 / 失效 / 格式异常 |
| 任一 Agent 的 Skill 根目录尚未创建 | 按未应用或部分应用聚合 |

状态优先级：

```text
冲突 > 未入库 > 失效 / 格式异常 > 项目路径不存在 > 部分应用 > 已应用 > 未应用
```

项目应用 Skill 时，目标为：

```text
targets = 当前项目 × 所有支持的 Agent
```

项目收回 Skill 时，也从当前项目的所有 Agent 收回。

## 5. 操作规则

| 操作 | 结果 |
|---|---|
| 从未应用拖到已应用 | 应用 Skill |
| 从已应用拖到未应用 | 收回 Skill |
| 双击未应用 Skill | 应用 Skill |
| 双击已应用 Skill | 收回 Skill |
| 选中后点向右箭头 | 应用 Skill |
| 选中后点向左箭头 | 收回 Skill |
| 部分应用再应用 | 补齐未启用的 Agent |
| 部分应用再收回 | 从所有 Agent 收回 |
| 冲突状态应用 | 弹确认，按主库优先覆盖 |
| 未入库状态 | 提供入库动作 |

普通应用和收回不弹窗，操作后显示回执。存在冲突、未入库、路径异常时，必须明确提示影响范围。

拖拽交互必须把正在拖动的对象作为独立 payload 处理，不依赖当前选中项或仅依赖 React 渲染状态。`dragstart` 写入 item id 和来源列，运行时同时用 ref 保存完整 item；`dragover` 只在目标列可接收时放行；`drop` 解析 payload 后复用箭头按钮同一条移动逻辑。这样用户拖动、双击和箭头操作的结果一致，且 WebView 事件顺序变化时不会静默失效。

拖拽、双击和箭头按钮必须共用同一组可移动性判断：

- 目标列是「已应用」时，只允许可应用、可补齐或可覆盖的条目移动。
- 目标列是「未应用」时，只允许已应用或部分应用的条目收回。
- 冲突、未入库、失效、格式异常等需要先确认或入库的状态，不能通过拖到「未应用」伪装成收回成功。

## 6. 组件设计

| 组件 | 作用 |
|---|---|
| `ObjectSidebar` | 渲染技能列表、技能组合、全局 Agent、项目、设置 |
| `SkillTransferView` | 当前 Agent / 项目的双栏应用工作台 |
| `SkillTransferColumn` | 未应用或已应用列表 |
| `SkillTransferRow` | 单个 Skill 行，支持点击、双击、拖拽、状态展示 |
| `TransferActionBar` | 左右箭头、批量应用、批量收回 |
| `ProjectAggregateStatus` | 汇总项目下多个 Agent 的状态 |

说明：Agent / 项目双栏工作台不使用右侧详情面板。需要看完整 `SKILL.md` 或主库路径时回到「技能列表」；应用视图只承载应用、收回、冲突覆盖和入库处理。

## 7. 数据流

现有后端已经返回 `agents`、`projects`、`skills`、`globalWorkspaces`、`projectWorkspaces`，并提供 `deploySkill`、`withdrawSkill`、`adoptSkillFromTarget`。

第一版优先在前端聚合，不新增数据库模型。

```text
全局 Agent:
globalWorkspaces[agentId].statuses
→ 拆分为未应用 / 已应用

项目:
projectWorkspaces[projectId + all agents].statuses
→ 按 skillId 聚合
→ 计算项目层状态
→ 拆分为未应用 / 已应用
```

## 8. 错误处理

| 场景 | 处理 |
|---|---|
| 应用失败 | 保持真实状态，顶部显示错误和下一步 |
| 部分成功、部分失败 | 显示成功、跳过、冲突、错误数量 |
| 冲突 | 弹确认，不默认覆盖 |
| 未入库 | 提供入库，不归入未应用 |
| Skill 根目录不存在 | 仍允许应用，操作时自动创建目录 |
| 项目不存在 | 左侧项目弱警告，主区域提示路径不可访问，并阻断应用 |

## 9. 验证标准

- `npm run build`
- `cargo check --manifest-path src-tauri/Cargo.toml`
- 桌面应用能启动。
- 点击 Trae / Codex / Claude Code 后，双栏状态正确，应用 / 收回只影响该 Agent。
- 点击项目后，双栏按项目聚合，应用 / 收回影响该项目全部 Agent。
- 冲突、未入库、失效和格式异常等问题出现在已应用侧。
- Skill 根目录不存在的未应用项可以应用；项目根路径不存在时必须阻断。
- 拖拽、双击和箭头按钮都能完成应用 / 收回。
- 常见桌面窗口尺寸下不重叠、不溢出。

## 10. 不做范围

- 不做项目内单 Agent 精细控制。
- 不做操作日志。
- 不做覆盖前备份。
- 不做自动扫描所有项目。
- 不做通用 Skill 市场浏览。
