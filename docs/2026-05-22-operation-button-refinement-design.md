# 操作按钮精致化设计

日期：2026-05-22

## 1. 背景

当前影子已经形成三栏工作台，但页面中大量操作都以 40px 文字按钮呈现。刷新、打开目录、编辑、删除、查看路径等工具动作与导入、分发、确认这类主动作视觉重量接近，导致界面显得粗、挤、操作层级不清。

本次调整目标不是追求纯图标化，而是让按钮承担更清楚的任务层级。

## 2. 核心判断

| 方案 | 判断 | 为什么 | 对用户的影响 |
|---|---|---|---|
| 全部按钮改成图标 | 不采用 | 新用户很难理解分发、入库、覆盖等本地文件动作 | 界面变轻，但学习成本和误操作风险上升 |
| 只缩小按钮尺寸 | 不足够 | 能减少拥挤，但刷新、删除、打开目录仍和主操作同权重 | 视觉略好，操作层级仍不清晰 |
| 主按钮保留文字，工具动作图标化 | 采用 | 既保留关键动作的可理解性，又降低次级操作噪音 | 工作台更精致，重复操作更快，关键动作仍清楚 |

## 3. 按钮层级

| 层级 | 用途 | 表达方式 |
|---|---|---|
| 主按钮 | 导入技能、分发、应用、保存、确认分发、完成初始化 | 深色文字按钮，保留图标 + 文案 |
| 次级文字按钮 | 取消、跳过、上一步、确认弹窗中的必要文字动作 | 浅色或透明文字按钮，尺寸收窄 |
| 工具按钮 | 刷新、打开目录、编辑、删除、移除、重新选择、添加对象、查看路径、展开正文 | 32px 图标按钮，使用 tooltip 与无障碍标签 |
| 筛选按钮 | 全部、有问题、已启用、未启用等状态筛选 | 30px 紧凑 chip，文字 + 数字 |
| 危险按钮 | 删除 Skill、删除组合、移除 Agent、移除项目 | 图标按钮 + 红色状态，点击后确认 |

## 4. 全局改动范围

- 顶部工具栏：刷新改为图标按钮；导入技能保留文字主按钮。
- 技能详情：分发保留文字主按钮；打开目录、删除改为图标按钮。
- 组合详情：应用组合保留文字主按钮；编辑、删除改为图标按钮。
- Agent / 项目应用详情：应用或入库保留文字主按钮；收回、打开位置改为图标按钮。
- 技能列表批量条：批量分发保留文字主按钮；清除选择改为图标按钮。
- 技能组合列表工具栏：新建组合改为图标按钮。
- 设置页：打开目录、初始化、添加、重新选择、移除等工具动作改为图标按钮。
- 抽屉和弹窗：关闭保持图标；确认、保存、取消等决策按钮保留文字。
- 路径与正文展开：查看路径和展开收起改为图标型微控件。

## 5. 验证标准

- `npm run build`
- `cargo check --manifest-path src-tauri/Cargo.toml`
- `npm run tauri:build`
- 顶部、详情、设置、弹窗、抽屉、初始化、空状态中的操作按钮都符合层级规则。
- 所有纯图标按钮都有 `aria-label` 和 `title`。
- 删除、移除等危险动作仍有二次确认。
- 常见桌面窗口尺寸下按钮不挤压文字，不导致列表或详情溢出。
