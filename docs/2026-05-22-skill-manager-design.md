# 技能中枢设计文档

日期：2026-05-22

## 1. 背景

本项目参考 `xingkongliang/skills-manager`，但不做通用开源工具克隆。参考项目覆盖 15+ coding tools，并包含 marketplace、presets、global workspace、project workspace、Git backup、activity log、CLI 等完整能力。

本项目面向个人本地使用，核心是把 Skill 作为知识资产统一管理，并清楚看到它们在 Trae、Codex、Claude Code 以及不同项目中的生效状态。

参考来源：

- `xingkongliang/skills-manager`：https://github.com/xingkongliang/skills-manager
- Impeccable 设计规范：https://impeccable.style/
- Impeccable product / brand 区分：https://impeccable.style/designing/
- Impeccable anti-pattern catalog：https://impeccable.style/slop/

## 2. 产品定位

产品中文名暂定为「技能中枢」。

一句话定义：

> 左侧选择管理范围，中间扫读技能和状态，右侧理解上下文并执行操作。

第一版目标：

- 做一个本地可视化桌面应用。
- 支持 Trae、Codex、Claude Code。
- 支持全局 Skill 和项目级 Skill 管理。
- 以 `~/.skills-manager` 作为统一主库。
- 通过软链接分发 Skill。
- 支持手动添加项目路径。

第一版不做：

- marketplace。
- 云同步。
- 多账号。
- 自动扫描所有项目。
- 覆盖前备份。
- 操作日志。
- 10+ Agent 的广泛适配。

## 3. 核心决策

| 维度 | 决策 |
|---|---|
| 产品形态 | 可视化桌面应用 |
| 技术方向 | Tauri 2 + React + TypeScript + SQLite |
| 支持 Agent | Trae / Codex / Claude Code |
| 主库路径 | `~/.skills-manager` |
| Skill 主体 | `~/.skills-manager/skills/<skill-name>` |
| 存量处理 | 导入主库，并保持原有 Agent / 项目可用 |
| 增量处理 | 直接进入主库，再通过桌面端分发 |
| 分发方式 | 软链接 |
| 冲突策略 | 主库优先覆盖目标同名 Skill |
| 备份 / 日志 | 第一版不做 |
| 项目管理 | 手动添加项目路径 |
| UI 视角 | 技能主库、全局应用、项目应用、技能套装、设置 |

## 5. 本地目录结构

```text
~/.skills-manager/
  skills/
    code-review/
      SKILL.md
      ...
    product-research/
      SKILL.md
      ...
  app.db
  config.json
```

| 目录 / 文件 | 作用 |
|---|---|
| `skills/` | 技能主库，每个子目录是一个 Skill |
| `app.db` | 当前状态索引，记录 Skill、Agent、项目、分发关系 |
| `config.json` | 主库路径、Agent 路径、界面偏好 |

Skill 目录至少包含：

```text
skill-name/
  SKILL.md
```

可选包含：

```text
skill-name/
  README.md
  scripts/
  assets/
  references/
```

## 6. 数据模型

| 概念 | 记录内容 | 用户看到什么 |
|---|---|---|
| `skills` | 主库里的技能名称、路径、描述、标签 | 技能主库列表 |
| `agents` | Trae / Codex / Claude Code 的全局技能目录 | 全局应用页 |
| `projects` | 用户手动添加的项目路径 | 项目应用页 |
| `deployments` | 某技能是否分发到某 Agent / 某项目 | 启用状态 |
| `settings` | 主库路径、软链接策略、UI 偏好 | 设置页 |

`deployments` 只作为当前状态索引，不作为历史日志。

## 7. 核心流程

### 7.1 首次启动：导入存量技能

```text
检测 Agent 技能目录
→ 展示发现结果
→ 标记：可入库 / 同名覆盖 / 疑似技能
→ 用户确认
→ 入主库并重建软链接
```

处理规则：

| 类型 | 处理 |
|---|---|
| 主库没有，Agent 里有 | 导入主库，然后把原目录替换成指向主库的软链接 |
| 主库已有，Agent 里同名 | 以主库为准，目标目录替换为软链接 |
| 目标本来就是软链接 | 校验是否指向主库，不是则重建 |
| 识别不到 `SKILL.md` | 显示为疑似技能，不自动处理 |

### 7.2 分发技能

```text
选择技能
→ 点击「分发」
→ 右侧抽屉选择目标：Trae / Codex / Claude Code / 项目
→ 展示将创建的软链接路径
→ 确认分发
```

分发结果示例：

```text
~/.codex/skills/code-review -> ~/.skills-manager/skills/code-review
```

项目级示例：

```text
<project>/.codex/skills/code-review -> ~/.skills-manager/skills/code-review
```

### 7.3 收回技能

收回只删除目标目录中的软链接，不删除主库技能。

| 场景 | 处理 |
|---|---|
| 目标是指向主库的软链接 | 删除软链接 |
| 目标是普通目录 | 不自动删除，显示为非本应用分发 |
| 主库技能本身 | 不受影响 |

### 7.4 处理冲突

状态为「冲突」时，右侧详情显示：

- 主库路径。
- 目标路径。
- 当前目标类型：普通目录 / 指向别处的软链接。
- 操作：用主库覆盖。

不提供复杂合并流程。第一版按主库优先。

### 7.5 删除技能

| 情况 | 处理 |
|---|---|
| 未分发 | 允许删除主库目录 |
| 已分发 | 提示先收回所有分发 |
| 用户强制删除 | 第一版不做 |

## 8. 状态定义

| 状态 | 含义 |
|---|---|
| 已启用 | 目标目录存在指向主库的软链接 |
| 未启用 | 目标目录没有该技能 |
| 未入库 | 目标目录有技能，但主库没有 |
| 冲突 | 目标目录有同名普通目录或指向别处的软链接 |
| 失效 | 软链接目标不存在 |
| 路径不存在 | Agent 或项目技能目录不存在 |

状态展示必须使用文字 + 徽标，不只依赖颜色。

## 9. UI / UX 设计

### 9.1 设计原则

采用「Codex app 式中文本地工作台」：

- 三栏结构。
- 列表优先。
- 状态优先。
- 少弹窗。
- 少卡片。
- 无装饰型 AI 风格。
- 所有高风险操作都在上下文里确认。

结合 Impeccable 的 product mode 原则：

- 设计服务任务，不做品牌展示。
- 先明确用户、场景、反例，再做界面。
- 迭代后要做可访问性、响应式、异常状态和文案打磨。

### 9.2 主界面

```text
┌────────────────────────────────────────────────────────┐
│ 技能中枢        搜索技能、项目、智能体应用       刷新  设置 │
├──────────┬────────────────────────┬────────────────────┤
│ 主库      │ 技能列表                │ 当前技能上下文        │
│ 全局      │ 名称 / 状态 / 启用范围    │ 摘要                 │
│ 项目      │ 标签 / 问题提示          │ SKILL.md 预览        │
│ 套装      │                        │ 分发位置             │
│ 设置      │                        │ 操作                 │
└──────────┴────────────────────────┴────────────────────┘
```

| 区域 | UX 目标 |
|---|---|
| 左栏 | 切换管理范围，不承载复杂信息 |
| 中栏 | 快速扫读所有技能和状态 |
| 右栏 | 解释当前对象，并给出下一步动作 |
| 顶部搜索 | 搜技能、项目、Agent、路径 |

### 9.3 页面设计

#### 技能主库

默认首页。

核心内容：

- 技能列表。
- 搜索和标签筛选。
- 技能详情。
- `SKILL.md` 预览。
- 启用位置。
- 分发 / 收回 / 打开目录。

#### 全局应用

按 Agent 查看全局技能状态。

```text
全局应用
[Trae] [Codex] [Claude Code]

code-review        已启用
product-research   未启用
local-only-skill    未入库
old-skill           冲突
```

#### 项目应用

只显示用户手动添加过的项目。

核心操作：

- 添加项目。
- 移除项目。
- 给项目启用技能。
- 停用项目技能。
- 查看某项目下 Trae / Codex / Claude Code 的 Skill 状态。

#### 技能套装

中文化的 preset。

示例：

| 套装 | 包含技能 | 用途 |
|---|---|---|
| 产品经理常用 | 调研、PRD、竞品分析 | 知识管理 / 产品设计 |
| 开发协作 | code-review、unit-test、debug | 开发项目 |
| 收尾归档 | neat-freak、文档同步 | 会话结束整理 |

#### 设置

设置独立成页，不塞进复杂弹窗。

内容：

- 主库路径。
- Trae / Codex / Claude Code 全局技能目录。
- 项目列表。
- 分发策略：固定软链接。
- 冲突策略：固定主库优先。

### 9.4 视觉系统

| 元素 | 规则 |
|---|---|
| 背景 | 温和浅色，略带灰度，不用刺眼纯白 |
| 主色 | 只用于选中态和主操作 |
| 状态色 | 绿色=已启用，橙色=冲突，红色=失效，灰色=未启用 |
| 字体 | 中文使用系统字体，路径和代码使用等宽字体 |
| 圆角 | 小圆角，偏工具感 |
| 阴影 | 尽量不用，用分割线、留白、层级表达结构 |
| 图标 | 线性图标，辅助识别，不代替文字 |

### 9.5 组件规则

| 组件 | 设计方式 |
|---|---|
| 技能列表 | 表格 / 紧凑列表，不用卡片网格 |
| 状态徽标 | 文字 + 小圆点 |
| 操作按钮 | 每屏只保留一个主按钮 |
| 分发面板 | 右侧抽屉，不用大弹窗 |
| 高风险确认 | 只有主库覆盖目标等操作才弹确认 |
| 空状态 | 一句话说明 + 一个动作 |
| 路径展示 | 等宽字体，可复制，可打开目录 |

### 9.6 明确避免的 UI 反模式

- 紫色渐变。
- 玻璃拟态。
- 发光边框。
- 卡片套卡片。
- 所有按钮都像主按钮。
- 复杂设置塞弹窗。
- 大段说明文。
- 灰字低对比。
- 无意义动效。
- 为了显得技术化而滥用等宽字体。

## 10. 错误处理

| 错误 | 处理 |
|---|---|
| Agent 路径不存在 | 在设置页提示配置路径 |
| 项目路径不存在 | 项目应用页标记路径不存在 |
| 软链接创建失败 | 显示目标路径和失败原因 |
| 目标同名冲突 | 显示冲突状态，提供用主库覆盖 |
| 主库 Skill 缺少 `SKILL.md` | 标记为格式异常 |
| 软链接目标不存在 | 标记为失效，提供清理 |
| 权限不足 | 显示需要手动授权或调整目录权限 |

## 11. 验证标准

进入实现后，每次迭代需要验证：

- 桌面应用能启动。
- 三栏主界面正常展示。
- 技能主库能扫描 `~/.skills-manager/skills`。
- Trae / Codex / Claude Code 的全局路径能配置和刷新。
- 手动添加项目能显示项目级状态。
- 分发能创建软链接。
- 收回只删除指向主库的软链接。
- 同名冲突按主库优先处理。
- UI 没有文字重叠、按钮溢出、低对比状态。
- 常见桌面窗口尺寸下可用。

## 12. 后续实现步骤

设计确认后，进入实现阶段前应先寻找适合的技能：

- UI 实现与体验检查：`ui-ux-pro-max`。
- 如果需要生成测试：`bits-unit-test-gen`。
- 如果需要代码审查：`bits-code-guard`。

若没有更适合的专用技能，再按普通开发流程实现。
