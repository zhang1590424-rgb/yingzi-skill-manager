# 技能列表与技能组合设计

日期：2026-05-22

## 1. 背景

当前界面和文档里同时存在「技能主库」「Skill 组」「技能套装」三套说法。它们对应的真实问题不同：

| 名称 | 本质 | 问题 |
|---|---|---|
| 技能主库 | 本地事实源路径 `~/.skills-manager/skills` | 偏实现概念，作为页面名会让用户以为在管理文件仓库 |
| 技能套装 | 旧 preset 命名 | 容易被理解成固定模板或市场套餐 |
| Skill 组 | 复合能力对象 | 中英混用，且入口藏在技能列表中，不利于组装 |

本轮设计把用户可见名称统一为：

- 「技能列表」：查看和管理单个 Skill。
- 「技能组合」：把多个 Skill 组装成可批量应用的能力单元。

主库仍是底层事实源概念，只在路径、冲突说明和技术规则里出现。

## 2. 设计目标

| 目标 | 为什么 | 对用户的影响 |
|---|---|---|
| 把「技能主库」页面改为「技能列表」 | 页面承载的是浏览和操作 Skill，不是让用户理解仓库模型 | 信息架构更贴近任务语言 |
| 把「技能套装 / Skill 组」统一为「技能组合」 | 组合强调可灵活组装，不暗示固定套餐 | 用户更容易理解它可以随时编辑成员 |
| 左侧独立保留「技能组合」入口 | 组合是一级管理对象，需要创建、编辑、删除和查看成员 | 不必在技能列表里找隐藏入口 |
| 在 Agent / 项目应用里支持技能和组合 | 应用场景里，用户关心启用能力，不关心能力来自单个 Skill 还是组合 | 可以单个启用，也可以整组启用 |
| 冲突时先预检再确认 | 分发会写入 Agent / 项目目录，错误覆盖成本高 | 操作前看清影响范围和风险 |

## 3. 信息架构

左侧导航调整为：

```text
技能中枢
  技能列表
  技能组合
  全局 Agent
    Trae
    Codex
    Claude Code
  项目应用
    <手动添加项目>
  设置
```

说明：

- 「技能组合」必须紧跟「技能列表」。
- 「全局 Agent」和「项目应用」仍是应用对象分组，不作为抽象页面入口。
- 「技能主库」不再作为页面标题或导航名称出现。

## 4. 技能列表

技能列表只处理单个 Skill。

核心能力：

- 查看所有主库 Skill。
- 搜索和状态筛选。
- 查看 `SKILL.md` 预览、主库路径、启用位置。
- 导入技能。
- 分发单个或批量选中的 Skill。
- 删除未启用 Skill。

第一版不在技能列表中默认折叠展示组合，避免用户混淆「单个 Skill 管理」和「组合编排」。

## 5. 技能组合

技能组合页用于灵活组装技能。

### 5.1 页面结构

```text
┌──────────────────────────────────────────────┐
│ 技能组合                         新建组合     │
├──────────────────────┬───────────────────────┤
│ 组合列表              │ 当前组合详情 / 成员     │
│ 飞书工作流  12 个技能  │ 名称、描述              │
│ 产品经理常用 5 个技能  │ 搜索技能、勾选成员       │
└──────────────────────┴───────────────────────┘
```

左侧为组合列表，右侧为组合详情和成员管理。仍保持工作台密度，不做卡片网格。

### 5.2 组合字段

| 字段 | 规则 |
|---|---|
| 名称 | 必填，保存时生成稳定 id；编辑名称不改变已有 id |
| 描述 | 可选，用于说明组合适用场景 |
| 成员 Skill | 从技能列表中多选，允许为空但不能应用 |
| 成员状态 | 显示存在、缺失、格式异常等基础状态 |

### 5.3 操作

| 操作 | 行为 |
|---|---|
| 新建组合 | 打开组合编辑弹窗或详情编辑态 |
| 编辑组合 | 可修改名称、描述、成员 |
| 删除组合 | 只删除组合配置，不删除主库 Skill |
| 应用组合 | 打开分发抽屉，选择 Agent / 项目目标 |
| 收回组合 | 从所选目标收回组合内所有已应用 Skill |

## 6. Agent / 项目应用

Agent 和项目应用页继续使用双栏：

```text
未应用                         已应用
技能 A                         技能 C
技能 B                         组合 X  部分应用 3/5
组合 Y                         组合 Z  已应用
```

列表项分两类：

| 类型 | 展示 | 操作 |
|---|---|---|
| Skill | 名称、描述、状态 | 应用、收回、入库、覆盖 |
| 技能组合 | 名称、描述、成员数、聚合状态 | 应用组合、收回组合、展开查看成员 |

组合行可展开查看成员，但成员仍可作为独立 Skill 单独应用或收回。

## 7. 状态聚合

技能组合状态由成员 Skill 在当前目标中的真实状态计算。

优先级：

```text
有问题 > 部分应用 > 已应用 > 未应用
```

细分规则：

| 成员状态 | 组合状态 |
|---|---|
| 全部未应用 | 未应用 |
| 全部已应用 | 已应用 |
| 一部分已应用、一部分未应用 | 部分应用 |
| 任一成员冲突、未入库、失效、路径不存在、格式异常、成员缺失 | 有问题 |

项目应用页需要按项目下所有 Agent 聚合：

- 所有 Agent 都已应用某成员，才算该成员已应用。
- 部分 Agent 已应用某成员，该成员为部分应用。
- 任一 Agent 有冲突或异常，该组合为有问题。

## 8. 冲突与兼容处理

### 8.1 分发策略

| 场景 | 处理 |
|---|---|
| 目标不存在同名内容 | 创建指向主库 Skill 的软链接 |
| 目标已是正确软链接 | 跳过 |
| 目标同名内容不是主库软链接 | 标记冲突，确认后按主库覆盖 |
| 目标软链接失效 | 标记失效，确认后可重建 |
| Agent / 项目下的 Skill 根目录不存在 | 应用时自动创建目标目录 |
| 项目路径不存在 | 阻断该目标，提示去设置修复路径 |
| 组合成员缺失 | 跳过缺失成员，提示编辑组合 |

### 8.2 组合预检

应用组合前必须展示：

- 将影响的目标数量。
- 组合内成员数量。
- 已应用会跳过多少个。
- 将新建多少个软链接。
- 冲突 / 未入库 / 失效 / 项目路径不存在数量。
- 最多展示前 4 个风险目标路径。

确认文案按风险变化：

| 风险 | 主按钮 |
|---|---|
| 无风险 | 应用组合 |
| 有冲突或失效 | 覆盖并应用 |
| 仅 Skill 根目录不存在 | 允许应用，提示会自动创建目录 |
| 项目路径不存在 | 禁用，并提示去设置处理 |

## 9. 数据与实现边界

第一版继续复用现有 `presets` / `preset_skills` 表，避免为改名引入数据库迁移风险。

产品和 API 命名统一使用：

- `SkillGroup` 在代码中逐步迁移为 `SkillComposition` 或保持类型名但 UI 文案统一为「技能组合」。
- 对外命令继续可复用现有 `upsert_preset`、`apply_preset`，但前端文案不再出现 `Skill 组`。

这次设计不要求立刻重命名 SQLite 表；后续如果需要组合排序、来源记录、嵌套组合，再单独设计迁移。

## 10. 错误处理

| 问题 | 页面反馈 |
|---|---|
| 组合名称为空 | 就地提示，不能保存 |
| 组合为空 | 可以保存，但应用按钮置灰 |
| 成员已从技能列表删除 | 组合详情显示缺失成员，保存时移除或要求重新选择 |
| 应用部分成功 | 回执展示变更、跳过、冲突、错误数量 |
| 收回遇到非主库软链接 | 不删除，标记跳过并提示不是本应用创建的链接 |
| 项目路径失效 | 项目应用页显示路径异常，阻断应用动作 |
| Skill 根目录尚未创建 | Agent / 项目应用页仍显示为未应用，应用时自动创建目录 |

## 11. UI 约束

- 保持 Codex app 式中文工作台。
- 使用三栏、双栏、抽屉和小弹窗，不做营销页。
- 不做卡片网格、紫色渐变、发光边框和装饰动效。
- 技能组合列表项和应用页组合行必须复用单个 Skill 的轻列表卡片基底：同样的小圆角、细边框、白色表面、选中态和右侧状态徽标。
- 组合卡片只在内容层区别于 Skill：展示成员数量、聚合状态和可展开成员；不要使用独立背景、重阴影或更大的卡片尺寸。
- 每屏只保留一个主操作。
- 图标按钮必须有 `aria-label` 或标题。
- 组合成员选择需要支持搜索，避免 Skill 数量增加后不可用。
- 文案使用「应用 / 收回」，不使用「安装 / 卸载」。

## 12. 验收标准

设计进入实现后，需要验证：

- 左侧导航显示「技能列表」「技能组合」，且组合在列表下面。
- 页面和主要文案不再出现「技能主库」作为导航或页面标题。
- 页面和主要文案不再出现「技能套装」。
- 技能组合页可以新建、编辑、删除组合。
- 组合可以从技能列表中灵活选择成员。
- Agent 应用和项目应用双栏能展示 Skill 与技能组合。
- 应用组合时只批量创建成员 Skill 的软链接，组合本身不软链接。
- 部分应用、冲突、项目路径不存在、成员缺失能正确聚合到组合状态。
- 有冲突时必须预检并确认；项目路径不存在时阻断；仅 Skill 根目录不存在时可直接应用。
- `npm run build` 和 `cargo check --manifest-path src-tauri/Cargo.toml` 通过。

## 13. 不做范围

- 不做组合嵌套组合。
- 不做组合市场、推荐或模板库。
- 不做覆盖前备份。
- 不做操作日志。
- 不做自动扫描所有项目。
- 不做项目内单 Agent 精细控制，项目应用仍默认影响项目下所有已配置 Agent。
