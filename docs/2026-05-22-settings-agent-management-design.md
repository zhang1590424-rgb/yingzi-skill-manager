# 设置页对象管理设计

日期：2026-05-22

## 1. 背景

左侧导航的职责是切换当前管理对象，不应该承担新增对象。项目和 Agent 都是应用范围配置，应该统一放在设置页维护。

同时，新增 Agent 的本质不是让用户理解目录协议，而是让系统识别一个 Agent 的全局 Skill 目录，并据此推断它在项目内的 Skill 目录。

## 2. 核心决策

| 维度 | 决策 | 为什么 | 对用户的影响 |
|---|---|---|---|
| 左侧新增入口 | 移除「项目应用」标题旁的新增按钮 | 左侧是对象导航，新增属于配置管理 | 日常切换项目时更安静，避免误触 |
| 设置页布局 | 设置页使用单列内容区，不使用三栏详情 | 设置项是维护表单，不需要列表 + 详情的阅读结构 | 项目、Agent、路径规则可以连续扫读 |
| 项目管理 | 设置页支持添加项目、移除项目 | 项目是用户手动纳入管理的对象 | 维护入口集中，不需要在左侧找小按钮 |
| Agent 管理 | 设置页支持添加 Agent、删除 Agent、更新全局目录 | 新 Agent 只需要被系统识别一次 | 可以扩展 Cursor、Windsurf 等 Agent |
| Agent 新增输入 | 用户只选择全局 Skill 目录 | 名称和项目目录规则可以从路径推断 | 不需要填写实现细节 |
| 项目级目录 | 从全局目录自动推断，不作为用户输入 | 大多数 Agent 约定是 `<项目根>/<agent根>/skills` | 项目分发路径稳定，设置心智更轻 |

## 3. Agent 推断规则

输入为一个全局 Skill 目录，例如：

```text
~/.codex/skills
```

系统按以下规则推断：

1. 如果目录最后一级是 `skills`，取它的上一级目录作为 Agent 根目录。
2. Agent 根目录保留原始目录名用于项目相对路径，例如 `.codex`。
3. 项目级目录固定为 `<Agent根目录>/skills`，例如 `.codex/skills`。
4. 显示名去掉前导点并标题化；命中内置映射时使用产品名。

内置映射：

| Agent 根目录 | 显示名 |
|---|---|
| `.codex` | Codex |
| `.trae` | Trae |
| `.claude` | Claude Code |

未命中映射时，`.cursor/skills` 显示为 `Cursor`，`.windsurf/skills` 显示为 `Windsurf`。

## 4. 设置页结构

```text
设置
  本地配置
    主库路径
    数据库
    配置文件

  Agent
    添加 Agent
    Codex
      全局目录: ~/.codex/skills
      项目目录规则: <项目根目录>/.codex/skills
      更新目录 / 删除

  项目
    添加项目
    工作
      路径: /Users/.../工作
      移除
```

### 4.1 布局细节

- 设置页不复用技能列表的「列表 + 详情」双栏布局，也不保留空的右侧详情区。
- 设置页主体在工作区内使用单列滚动内容，宽度优先服务长路径扫读；超宽窗口下可以保留自然留白，但不能出现中间分割线。
- Agent 和项目配置行统一为两段结构：左侧是名称、状态和路径信息，右侧是启用、重新选择、移除等操作。
- 路径信息使用标签 + 等宽路径文本展示，允许长路径在单元内自然换行，不挤压操作按钮。

## 5. 错误与保护

| 场景 | 处理 |
|---|---|
| 选择的目录不存在 | 阻断新增或更新 |
| 选择的目录不是 `skills` 目录 | 允许，但推断为 `<最后一级目录>/skills` 可能不稳定；第一版先提示错误，要求选择 Skill 目录 |
| 重复添加同一个 Agent 根目录 | 使用同一个 Agent id 更新配置，不生成重复项 |
| 删除 Agent | 只从影子管理列表移除，不删除任何真实目录和 Skill |
| 删除最后一个 Agent | 阻断删除，避免项目页没有可分发目标 |

## 6. 验证点

- 左侧「项目应用」没有新增按钮。
- 设置页为单列内容区。
- 设置页可以添加和移除项目。
- 设置页可以通过选择全局 Skill 目录添加 Agent。
- 新增 Agent 后左侧全局 Agent 分组和项目聚合都会出现对应对象。
- 项目级分发路径使用自动推断的 `<项目根>/<agent根>/skills`。
