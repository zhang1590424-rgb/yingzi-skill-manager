# 工作台信息结构与视觉分割收敛设计

日期：2026-05-23

## 1. 背景

当前设置页、技能组合页和顶部区域暴露出同一个问题：页面结构更多来自实现惯性，而不是用户任务。

- 设置页把 Agent、项目、本机路径和健康问题放成多个平级区块，用户不容易判断哪些是“应用位置”，哪些是“本机数据”。
- 技能组合页在没有组合时仍保留详情区域，形成没有对象却有详情的空壳。
- 顶部、列表、详情和区块之间叠加了多层分割线，层级感依赖线条而不是内容、留白和状态。

本次改版目标不是增加新能力，而是统一工作台语法：默认看列表或对象，点击后再打开可关闭详情；操作放回对象卡片；视觉分割只服务真实层级。

## 2. 核心决策

| 维度 | 决策 | 为什么 | 对用户的影响 |
|---|---|---|---|
| 设置页主结构 | 第一层改为「应用位置」 | Agent 和项目本质上都是 Skill 被应用到哪里，不应像两套无关配置 | 用户先看到影子正在管理哪些位置，再理解路径细节 |
| 设置页本机路径 | 下沉为「本机数据」 | 主库、数据库、配置文件是事实源和技术支撑，不是日常首要任务 | 页面从“路径配置表”变成“管理对象工作台” |
| 健康总览 | 仅有问题时出现在应用位置上方 | 没问题时不占位，有问题时优先处理 | 日常页面更安静，异常时有明确入口 |
| 技能组合页默认态 | 对齐技能列表：默认是组合列表 | 没有选中对象时，详情区域没有存在理由 | 没有组合时不再出现空右栏 |
| 技能组合详情 | 点击组合后打开可关闭详情 | 详情是用户动作触发的阅读面板，不是常驻布局 | 用户可以回到纯列表状态，页面更轻 |
| 技能组合操作 | 操作放在组合卡片右侧，悬停 / 聚焦时出现 | 现有技能卡片已经建立了状态与操作互换的语法 | 组合和技能的使用方式一致，不需要重新学习 |
| 顶部分割线 | 减少常驻边框，改用背景、留白和局部边界表达层级 | 线条过多会把工作台切碎，尤其在顶部和面板交界处 | 页面更安静，用户注意力回到对象和状态 |

## 3. 设置页信息结构

### 3.1 页面顺序

```text
设置
  健康总览（仅有问题时出现）

  应用位置
    全局 Agent
      Codex / Trae CN / Claude Code / Aiden / 手动添加 Agent
    项目
      手动添加的项目路径

  本机数据
    技能主库路径
    数据库路径
    配置文件路径
```

### 3.2 应用位置

「应用位置」是设置页的第一主区块。

- 全局 Agent 和项目都属于同一类对象：Skill 的应用目标。
- 区块内再分成「全局 Agent」和「项目」两个小组，避免把 Agent 和项目做成两个视觉重量相同的大模块。
- 添加 Agent、添加项目是该区块的局部操作，放在对应小组标题或工具区。
- Agent 行展示：名称、启用状态、全局 Skill 目录、项目目录规则、重新选择、移除。
- 项目行展示：项目名、路径状态、项目路径、移除。

### 3.3 本机数据

「本机数据」放在应用位置之后，视觉重量低于应用位置。

- 主库路径保留，因为它解释影子的唯一事实源。
- 数据库和配置文件保留，但不作为页面第一屏核心。
- 打开主库目录、重新打开初始化引导等工具操作保留为图标按钮。

### 3.4 健康总览

健康总览仍复用现有问题聚合逻辑，但不作为固定装饰。

- 有问题时显示在「应用位置」上方。
- 无问题时不渲染，不留下空白。
- 问题按钮继续打开问题处理抽屉。
- 文案必须指向下一步，不使用宽泛排查建议。

## 4. 技能组合页结构

### 4.1 页面状态

技能组合页对齐技能列表的工作台语法。

| 状态 | 页面结构 |
|---|---|
| 没有组合 | 只显示整页空态和「新建组合」入口，不挂载详情面板 |
| 有组合但未选中 | 只显示技能组合列表 |
| 点击组合 | 右侧详情面板出现，列表保留选中态 |
| 关闭详情 | 回到纯列表状态 |
| 搜索无结果 | 显示当前筛选无匹配，不打开详情 |

### 4.2 组合卡片

组合卡片复用技能卡片的基础交互，而不是单独设计一套行样式。

默认态展示：

- 组合名称。
- 描述；没有描述时展示成员名称摘要。
- 成员数。
- 健康状态：成员正常、空组合、成员缺失、成员有问题。

悬停或键盘聚焦时，右侧状态区域切换为操作：

- 应用组合。
- 编辑组合。
- 删除组合。

操作规则：

- 点击卡片主体打开详情。
- 点击操作按钮不触发详情切换。
- 空组合禁用「应用组合」，保留编辑和删除。
- 有缺失成员时允许应用，但应用前继续提示会跳过缺失成员。
- `focus-within` 必须和 hover 一样显示操作，保证键盘可用。

### 4.3 空态

空态只服务一个问题：用户下一步该做什么。

空态内容：

- 标题：`还没有技能组合`
- 说明：`把多个技能组织成一个可批量应用的组合。组合不会生成独立 Skill，只会批量处理成员。`
- 主操作：`新建组合`

不展示推荐模板、示例组合列表或 marketplace 入口，避免偏离第一版边界。

### 4.4 组合详情

组合详情是临时阅读面板，必须可关闭。

详情保留三类信息：

1. 顶部摘要：名称、描述、成员数、健康状态。
2. 成员清单：存在、缺失、格式异常等状态。
3. 操作区：应用组合、编辑、删除。

「组合规则」不再作为独立大区块，只保留一句轻提示：

```text
组合本身不会被软链接；应用或收回组合时，只批量处理成员 Skill。
```

## 5. 顶部与分割线规则

### 5.1 总原则

视觉分割只在用户需要判断空间边界时出现。

- 主导航和主工作台之间保留必要边界。
- 列表和详情之间只在详情打开时保留边界。
- 纯列表状态下不显示右侧边界。
- 页面顶部不叠加多条横线；优先用背景、留白、标题层级表达区域。
- 区块内部不用连续 `border-top` 堆叠层级，改为行间距、浅背景、局部边界。

### 5.2 需要保留的分割

| 位置 | 处理 |
|---|---|
| 左侧导航与主工作台 | 保留弱竖向边界 |
| 列表与详情 | 仅详情打开时保留弱竖向边界 |
| 弹窗、抽屉头部 | 保留边界，帮助理解浮层结构 |
| 设置页行内项目 | 保留轻边界或背景分组，避免长路径混在一起 |

### 5.3 需要削减的分割

| 位置 | 处理 |
|---|---|
| 顶部搜索栏下方常驻重线 | 降低存在感，避免和列表标题线叠加 |
| `pane-header` 的强边界 | 视页面状态调整，纯列表页减少边框重量 |
| 设置页大区块之间连续横线 | 改为分组标题、留白和局部容器 |
| 组合空态右侧边界 | 取消，因为没有详情面板 |

## 6. 实现范围

本次实现应覆盖：

- 设置页 JSX 结构调整为「健康总览 / 应用位置 / 本机数据」。
- 设置页样式调整，减少大区块平级边框。
- 技能组合页的 `workspace` 状态与技能列表保持一致：未选中时列表独占，选中后详情出现。
- `PresetList` 卡片交互对齐 `SkillList`：状态和操作在同一区域互换。
- 组合空态改为整页空态。
- 组合详情增加关闭入口，并降低规则说明视觉重量。
- 顶部、面板、区块的分割线按本设计统一收敛。

不在本次实现：

- 不新增组合模板或推荐示例。
- 不新增 marketplace 入口。
- 不改变技能组合的数据模型。
- 不改变软链接分发和收回规则。
- 不重写 Agent / 项目的底层扫描逻辑。

## 7. 验证点

- 设置页第一主区块为「应用位置」，Agent 和项目在同一语义下展示。
- 设置页无健康问题时不显示健康总览。
- 技能组合为空时不显示右侧详情栏。
- 技能组合有数据但未选中时只显示列表。
- 点击组合后详情出现，点击关闭后回到纯列表。
- 组合卡片默认显示状态，悬停和键盘聚焦时显示操作。
- 卡片操作不会误触发详情打开。
- 空组合不能应用，但可以编辑和删除。
- 顶部和页面区块不出现重复、过重的横向分割线。
- 常见桌面窗口尺寸下文本不重叠、不溢出。
