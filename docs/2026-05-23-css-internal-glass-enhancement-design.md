# CSS 内部玻璃增强设计

日期：2026-05-23

## 1. 背景

当前侧边栏、初始化进度栏和欢迎页影子层已经使用半透明背景与 `backdrop-filter`，但底层背景过于平整，视觉上接近实色浅灰。用户感知不到「类似 Codex 的半透明玻璃状」。

本次目标是在不引入原生窗口透明和 macOS vibrancy 的前提下，通过 CSS 内部层次增强玻璃感。

## 2. 核心决策

| 决策 | 为什么 | 对用户的影响 |
|---|---|---|
| 只做 CSS 内部增强 | 原生透明窗口会带来可读性、拖拽、性能和平台兼容成本 | 更稳，不改变桌面应用窗口行为 |
| 给玻璃层背后增加低对比材质纹理 | `backdrop-filter` 需要可被模糊的底层变化，否则看起来像实底 | 侧栏和引导 rail 的半透明感更明显 |
| 提升非焦点区的透明边界和内高光 | Codex 式材质感来自边界、阴影、内高光和背景漂移的组合 | 视觉更轻，但仍能识别导航结构 |
| 焦点区域继续保持实底 | 列表、表单、按钮和主工作区需要稳定阅读 | 不牺牲长期使用的清晰度 |

## 3. 范围

- 增强 `.app-shell` 背景层次，让主侧边栏下方有可被模糊的细微变化。
- 增强 `.sidebar` 的半透明材质、内高光和低对比边界。
- 增强 `.onboarding-shell`、`.onboarding-shell.welcome` 的背景层次。
- 增强 `.onboarding-rail` 的半透明材质。
- 增强 `.shadow-plane` 的玻璃观感，但不加彩色发光、装饰球或强渐变。

## 4. 不做范围

- 不开启 Tauri 原生透明窗口。
- 不把主内容区、列表行、表单、按钮、抽屉主体、弹窗主体玻璃化。
- 不使用紫色渐变、彩色光、bokeh、发光边框。
- 不改布局、不改业务逻辑。

## 5. 验证标准

- `npm run build` 通过。
- 桌面常见尺寸下侧栏和初始化引导不重叠、不溢出。
- 侧栏和初始化 rail 能明显看出半透明材质，但文字和 active 项仍清晰。
