# Agent 选择与项目分发范围修正设计

日期：2026-05-26

## 1. 问题

初始化阶段如果用户没有选择某个 Agent，进入工作台后该 Agent 仍可能出现在左侧列表；在项目级分发 Skill 时，也可能继续为该 Agent 创建项目内的 Skill 目录。

根因有两层：

- 左侧导航使用 `agents` 全量渲染，没有按 `enabled` 过滤。
- 初始化 Agent 步骤的「跳过」没有保存当前勾选状态；如果用户取消勾选后跳过，后端仍保留默认启用状态。

## 2. 决策

| 维度 | 决策 | 为什么 | 对用户的影响 |
|---|---|---|---|
| 左侧 Agent 列表 | 只展示已启用 Agent | 左侧是工作台范围，不是完整配置清单 | 未选择的 Agent 不再干扰日常分发 |
| 设置页 Agent 列表 | 继续展示全部 Agent，并保留启停开关 | 设置页负责维护配置，不能隐藏可恢复对象 | 用户仍能重新启用初始化时未选择的 Agent |
| 初始化跳过 | 保存当前勾选状态后再进入下一步 | 用户已经在当前页表达了管理范围，跳过不应丢弃选择 | 取消勾选后即使点「跳过」也会生效 |
| 项目分发目标 | 只展开已启用 Agent，并由后端拒绝停用 Agent 目标 | 前端过滤负责体验，后端兜底避免误写目录 | 不会为未选择 Agent 创建 `.xxx/skills` 目录 |
| 项目根路径 | 后端再次校验项目根目录存在 | 项目根不存在时创建目录属于误写入 | 即使前端状态过期，也不会创建不存在的项目根 |

## 3. 验证

- 初始化取消勾选某个 Agent 后进入工作台，左侧不展示该 Agent。
- 在设置页停用 Agent 后，左侧立即移除该 Agent，但设置页仍保留该行。
- 项目分发 Skill 时，预检目标数量只按已启用 Agent 计算。
- 即使收到停用 Agent 的分发请求，后端也不创建对应项目 Skill 目录。
