# 本地 Skill 导入弹窗设计

日期：2026-05-22

## 1. 背景

当前导入弹窗同时放了「本地文件夹」和「市场链接」。市场链接可用性差，依赖公司网络、登录状态和外部安装器，失败后用户很难判断是链接、权限、网络还是安装器问题。它不应该占据「导入技能」这个最高频入口。

## 2. 设计决策

| 决策 | 为什么 | 对用户的影响 |
|---|---|---|
| 移除市场链接入口 | 第一版核心任务是稳定管理本地 Skill 主库，市场链路外部依赖太多 | 导入弹窗不再出现来源切换和 URL 表单 |
| 保留本地文件夹导入 | 主库事实源是本地目录，本地 Skill 可直接校验 `SKILL.md` | 用户只要选一个 Skill 文件夹即可入库 |
| 使用整块选择区 | 导入行为本质是“选择一个目录”，不需要按钮和说明分散在不同位置 | 点击区域更大，意图更直接 |
| 导入后不自动分发 | 入库和启用是两个不同决策 | 新 Skill 先进入主库，再由用户决定分发到哪些 Agent 或项目 |

## 3. 交互规格

```text
点击「导入技能」
→ 弹窗只展示一个本地文件夹选择区
→ 点击选择区打开系统文件夹选择器
→ 选择包含 SKILL.md 的文件夹
→ 复制到 ~/.skills-manager/skills
→ 关闭弹窗并选中新 Skill
```

文案：

- 主文案：`点击选择技能文件夹`
- 辅助文案：`需要包含 SKILL.md`
- 补充文案：`导入后进入主库，不会自动分发`

## 4. 验收标准

- 导入弹窗不再出现「市场链接」标签、URL 输入框、版本号输入框和安装按钮。
- 整块选择区可点击，禁用状态下不可点击。
- 弹窗保留关闭、取消、Esc 关闭和焦点管理。
- `npm run build` 通过。
