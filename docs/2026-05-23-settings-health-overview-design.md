# 设置页健康总览设计

## 背景

设置页已经能列出当前问题，但只是静态文本。用户看到失效软链接、未入库 Skill 或冲突后，还要自己去猜下一步在哪处理，形成断点。

## 目标

- 设置页顶部恢复「健康总览」，作为问题处理入口。
- 保留问题聚合口径：按冲突、失效链接、路径不存在、未入库、格式异常分类。
- 点击分类后打开问题处理抽屉，复用现有忽略、查看和处理能力。

## 交互规则

- 「健康总览」放在设置内容顶部，高于本地配置。
- 有问题时展示分类按钮和数量；无问题时不额外占用空间。
- 失效链接支持「处理」：删除指向不存在位置的软链接。
- 未入库支持「处理」：导入主库，并把目标位置改为指向主库的软链接。
- 冲突支持「处理」：按主库版本覆盖目标同名内容。
- 暂不能自动修复的问题保留「查看」和「忽略」，避免误删真实文件。

## 用户影响

- 用户从设置页看到问题后可以直接完成收尾，不需要在全局 Agent、项目应用和技能列表之间来回找入口。
- 健康问题与具体对象仍保持关联；点击「查看」会定位到对应 Agent 或项目下的技能状态。
- 对真实文件有破坏风险的动作继续走确认弹窗。
