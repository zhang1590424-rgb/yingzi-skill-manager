# 影子设计约束

## 设计模式

采用 Impeccable 的 Product lane：服务任务，不做品牌展示。产品气质是安静、可信、上下文连续、适合长时间工作的中文本地工作台。

## 品牌与图标

- 产品正式名为「影子」。
- 品牌隐喻是“主库 Skill 作为实体，在不同 Agent 和项目中生成影子”。
- 图标使用实体与多个偏移投影表达分发关系，不使用暗黑、神秘、赛博、发光或装饰型 AI 风格。
- 应用内品牌区必须优先传达当前工具身份，不扩展成营销式标语。

## 布局

- 默认固定三栏：左侧管理范围，中间列表，右侧上下文详情；设置页是维护表单，使用单列内容区。
- 左侧导航使用「技能列表」和「技能组合」作为两个相邻入口，不再把「技能主库」作为页面名。
- 应用主体锁定在单个桌面视口内：左侧栏不随技能列表滚动，中间列表和右侧详情在各自栏位内独立滚动。
- 左侧导航按自然高度紧凑排列，空余高度留在导航区末尾，不把菜单组拉散。
- 切换视图、Agent、项目时，右侧详情必须同步到当前范围。
- macOS 原生标题栏不显示可见窗口标题；产品名只在侧栏品牌区、引导页和应用元信息中保留，避免顶部系统操作区旁边重复出现「影子」。
- 设置独立成页，不塞进复杂弹窗；项目和 Agent 的新增、删除都放在设置页，不放在左侧导航。
- 设置页必须拆成三个视觉一致的同级管理区块：「我的 Agent」「我的项目」「技能根目录」。Agent 是全局运行环境目录，项目是手动纳入管理的项目路径，技能根目录以主库为本地事实源并展示关联数据库和配置文件，三者不能混在同一个容器里。
- 分发使用右侧抽屉，导入、组合编辑和确认使用小弹窗。
- 新增 Agent 时只选择全局 Skill 目录，名称和项目级目录由系统从路径自动推断。
- 技能组合页用于创建、编辑、删除组合，以及从技能列表中搜索和勾选成员。
- Agent / 项目应用页需要同时展示单个 Skill 与技能组合，组合行可展开查看成员。

## 视觉

- 冷中性浅灰背景，主操作使用近黑或深灰。
- 状态色低饱和，必须有文字，不只依赖颜色。
- 小圆角、细边框、轻底色；常规区域不用大阴影。
- 允许 Skill 条目使用轻卡片式列表行；不使用卡片网格、市场式大卡片、紫色渐变、发光边框、装饰型玻璃拟态、卡片套卡片。
- 技能组合条目必须复用 Skill 条目的轻列表卡片语言，只通过成员数、聚合状态和展开控件表达差异，不另起一套视觉风格。
- 使用背景材质层级表达注意力：非当前关注区域可以使用克制半透明、轻模糊和低对比；当前操作区、列表、表单、按钮、弹窗主体必须保持清晰实色和足够对比。透明效果只用于降低背景噪声，不能演变成装饰型玻璃拟态。

## 字号与密度

- 工作台正文以 13px 为主。
- 列表说明、路径、标签以 11.5-12px 为主。
- 页面标题控制在 16-17px。
- SKILL.md 预览可以略大，但不能压过当前操作区。
- 技能列表条目可以略高以容纳描述和标签；应用双栏条目保持紧凑，优先服务应用 / 收回操作。

## 交互

- 每屏只保留一个主按钮。
- 删除、覆盖等高风险操作必须确认。
- 分发前展示最终软链接路径和目标状态。
- 导入技能只保留本地来源，弹窗主体使用整块选择区，文案直接说明可选择包含 `SKILL.md` 的文件夹或 `.zip` 压缩包。
- 操作反馈使用轻量悬浮 toast，不插入页面布局，出现和消失都不能造成页面高度跳变。
- 弹窗和抽屉内触发的错误也使用悬浮 toast，层级必须高于遮罩和浮层。
- 弹窗和抽屉必须可键盘关闭，并提供明确焦点状态。
- 首次启动使用非阻塞初始化清单，不做强制教程；用户可以关闭清单继续使用。
- 初始化 Agent 勾选结果就是工作台管理范围：未勾选或已停用的 Agent 不出现在左侧「我的 Agent」，也不参与项目级 Skill 分发展开；它们只保留在设置页中，方便稍后重新启用。
- 空状态必须说明当前区域的下一步动作；没有技能、组合或项目时不能只显示“没有数据”。
- Agent Skill 根目录不存在时允许应用并自动创建；项目根路径不存在时必须阻断应用并引导到设置处理。
- Agent 或项目下的 Skill 目标目录可自动创建时，不在应用列表和问题筛选中暴露为用户待处理状态；只有冲突、未入库、失效、格式异常、项目根路径不可访问这类需要用户决策或修复的状态才占用提示位。
- 首页不再常驻健康总览；异常信息优先回到具体列表筛选、状态徽标和对象详情中表达，避免非主线内容抢占工作台第一屏。
- 「未入库」只用于目标位置存在可导入 Skill 实体的情况；断裂软链接必须归类为「失效」，不能进入存量 Skill 入库流程。
- 初始化阶段的存量 Skill 入库必须以重新扫描结果为准；只要批量入库出现错误，就停留在当前步骤展示原因，不能直接进入工作台。
- 初始化后出现的健康问题必须提供明确收尾动作：忽略后同一问题不再提示，处理则按问题类型一次性修复并重新扫描。
- Agent / 项目应用页的双栏已经表达未应用和已应用，顶部不再常驻展示全部、未应用、已应用等重复筛选；仅在存在真实问题时显示非零问题筛选。
- 加载本地状态失败时必须提供刷新重试入口，不能让主工作区空白。
